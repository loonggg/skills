# 构图模式库

## 8 种结构类型

### 1. Workflow（工作流）
输入 → 处理 → 输出。材料从一端进，从另一端出。
适合：流程、pipeline、步骤链。

### 2. System Local（系统拆解）
3-5 个模块组成一个系统，模块之间有关联线。
适合：架构、系统组成、模块关系。

### 3. Before/After（前后对比）
左边混乱/有问题，右边稳定/解决后。中间用箭头或分隔线。
适合：优化、改造、效果对比。

### 4. Character State（状态变化）
2-4 个状态，同一个主体在不同状态下的表现。
适合：生命周期、阶段变化、状态机。

### 5. Concept Metaphor（概念隐喻）
一台大机器 / 一个装置，用物理隐喻表达抽象概念。
适合：核心原理、机制解释、抽象概念具象化。

### 6. Method Layers（方法分层）
分层盒子，从上到下或从内到外。
适合：层次结构、协议栈、方法分解。

### 7. Map Route（路线图）
弯曲路径，沿途经过关键节点。
适合：成长路径、发展历程、旅程。

### 8. Mini Comic（迷你漫画）
2-4 个场景格子，用连续画面讲故事。
适合：叙事、案例、时间线故事。

## 隐喻生成方法

1. **抽象 → 物理动作**：把抽象概念变成可以用手操作的动作
2. **系统 → 低科技物件**：把软件系统变成纸箱、抽屉、老机器、漏斗、秤、邮箱
3. **让墨迹执行动作**：墨迹是干活的工人，不是看客

## 物件池

纸箱、抽屉、老机器、漏斗、秤、邮箱、门、井、梯子、管道、
传送带、筛子、锅炉、按钮、闸门、绳索、滑轮、水龙头、桶

## 动作池

拉、搬运、塞入、舀、按压、称重、缝、切、拧、看守、
推、接住、拆解、标记、回收、倒、摇、踩、翻、堆叠

## 反抄袭规则

示例仅供风格校准。每次必须发明全新隐喻。如果发现自己在套用已有构图模式的具体元素，停下来重新想。
