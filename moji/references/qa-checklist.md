# QA 自检清单

## 必须通过

- [ ] 16:9 比例（1280×720px）
- [ ] 纯白干净背景，无纹理/渐变/阴影
- [ ] 墨迹在场，且执行核心动作（不是旁观者/吉祥物）
- [ ] 隐喻是全新的（非复用示例或常见套路）
- [ ] 荒诞 / 有趣 / 有创意
- [ ] 主体占比 < 60%
- [ ] 只表达一个核心结构（不堆砌信息）
- [ ] 5-8 个中文标注，每个 2-8 字，可读
- [ ] 颜色使用正确：黑（结构）+ 橙（流动）+ 红（关键）+ 蓝（可选辅助）
- [ ] 无左上角标题

## 失败信号

出现以下任一情况需重做：

- 左上角大标题 → 删掉，用标注代替
- 墨迹象吉祥物/可爱 → 重写墨迹动作
- PPT 感 → 打破规整方框，加入手绘不规则感
- 元素过多 → 砍到只剩核心结构
- 大段文字 → 压缩到 2-8 字标注
- 有纹理/渐变背景 → 改为纯白
- 像 App 界面 → 用物理隐喻替代
- 中文标注不可读 → 增大字号或简化文字
- 太规整 → 加入手绘抖动
- 和已有作品太像 → 换隐喻

## 修正方法

| 问题 | 修正方向 |
|------|---------|
| 太平淡 | 加入荒诞感：墨迹的动作更夸张，隐喻更出人意料 |
| 太复杂 | 砍掉次要元素，只留核心结构 |
| 太可爱 | 让墨迹更面无表情，动作更"干活"更少"卖萌" |
| 太 PPT | 用手绘线条替代规整方框，加入不规则感 |
| 太像旧作 | 完全换一个隐喻方向 |
| 文字错误 | 检查每个标注的用字和字数 |

## 交付标准

读者看到图的第一反应应该是"有点奇怪"，然后一秒钟内理解隐喻在说什么。
